## 活动错误档案

### E01: 忘记关闭文件句柄导致句柄泄漏
- **关联知识点**：[[02-知识点地图#K01]]
- **发现时间**：2026-06-21
- **触发场景**：在编写循环打开 2000 个小文件合并内容时，程序抛出 `OSError: [Errno 24] Too many open files`。
- **根因分析**：在每次打开文件读取内容后，直接通过 `continue` 或返回跳出，未能执行 `f.close()`，导致操作系统持有的文件句柄数量耗尽。
- **纠正代码**：
  ```python
  # 错误做法：未 close
  for path in file_paths:
      f = open(path, 'r')
      data = f.read()  # 如果中间抛出异常或没有 close，句柄累积
      
  # 正确做法：
  for path in file_paths:
      f = open(path, 'r', encoding='utf-8')
      try:
          data = f.read()
      finally:
          f.close()
  ```
- **验证结果**：在 2026-06-21 的练习中，已能准确找出代码段中的句柄泄漏错误并使用 finally 块纠正。
