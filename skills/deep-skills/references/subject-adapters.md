# Subject Adapters

Read this file whenever teaching a technical, mathematical, scientific, or computational topic. Use the relevant adapter in addition to the universal mastery protocol.

## Mathematics

For each key point, check:

- definition in the learner's words;
- meaning of each symbol;
- assumptions and domain;
- one worked derivation with justification for each nontrivial step;
- a near problem, a changed-form problem, and an error analysis;
- a counterexample or boundary where appropriate.

Do not assess only final numeric answers. Ask why a step is legal, what condition is used, whether a different method works, and how the result changes if an assumption changes.

Useful artifact types: derivation sheet, proof outline, formula card with conditions, error catalog, small modeling problem.

## Physics and engineering science

For each problem or model, require:

1. system boundary and target quantity;
2. assumptions and what is ignored;
3. model/law selection;
4. units or dimensions;
5. limiting-case or order-of-magnitude check;
6. interpretation of the result in the real situation.

Include model-building, diagram/graph interpretation, experiment or simulation design, uncertainty/error reasoning, and changed-condition transfer.

## Chemistry

Require more than fact recall. Check structure–property relationships, conditions, mechanisms or reaction paths, competing explanations, expected observations, and experimental controls.

Practice should include: structure classification, condition selection, phenomenon prediction, mechanism explanation, error diagnosis, and an experiment/design task when appropriate.

## Biology and life science

Map mechanisms as ordered causal chains. Check process order, molecular/cellular level distinctions, feedback/control relationships, experimental controls, graph interpretation, and new-case application.

Useful tasks: reconstruct a pathway, distinguish correlation from mechanism, design a controlled experiment, explain a perturbation, and identify an overgeneralized conclusion.

## Programming and computer science

A programming point is not mastered by reading code. Require at least three independent evidence forms across a unit:

- write a small function or data transformation;
- complete or modify code;
- diagnose and fix a bug;
- add or improve a test;
- explain execution, data flow, or a call path;
- adapt a solution to a changed requirement;
- justify an interface, complexity, or tradeoff.

Use this progression:

1. smallest executable demonstration;
2. local function exercise;
3. combine two concepts;
4. debug a deliberate fault;
5. capstone skeleton;
6. learner implements key logic;
7. tests or manual verification;
8. learner explains behavior;
9. changed requirement;
10. reflection on tradeoffs.

Never claim code runs or tests pass without host confirmation. Keep potentially harmful security, scraping, credential, or system automation tasks within lawful and authorized boundaries.

## Statistics, machine learning, and data science

Check the data-generating context, variable definitions, assumptions, metric choice, model fit, interpretation, and limits of inference. Include data leakage, overfitting, spurious correlation, selection bias, uncertainty, and causal overclaim checks when relevant.

Practice should include selecting a method from a scenario, explaining assumptions, interpreting an output, finding an invalid conclusion, and a small analysis/design task.

## Economics and quantitative social science

Make assumptions and ceteris paribus conditions explicit. Require model interpretation, diagrams or equations, comparative statics, counterexamples, empirical limits, and policy tradeoffs. Distinguish normative claims from descriptive claims.

## Cross-subject capstone design

A capstone must list:

- target outcome;
- `Kxx` coverage matrix;
- learner-produced artifact;
- acceptance criteria;
- test, validation, proof, measurement, or review method;
- likely failure modes;
- a changed-condition extension task;
- an explanation defense where the learner justifies choices.
