# ADR 0001: Mastery Storage Soft Degradation (存储依赖软降级决策)

## 状态
已通过

## 上下文
我们的 Mastery Learning 技能不仅要在具备完整本地读写权限的个人知识库环境（如 Obsidian 本地 Vault）中运行，还要能够在网页端、云端笔记本（如 Notion、飞书）以及无文件系统读写权限的纯聊天窗口（Chat-only）中工作。
如果强制要求本地文件系统写入，会导致无法在轻量级聊天场景或云场景中使用，从而增加学生进入学习的门槛。

## 决策
我们决定将学习与评估逻辑设计为**对存储机制的“软依赖 (Soft Dependency)”**，并引入三层降级退化方案：

1. **完全本地模式 (Full Mode)**：
   - **条件**：经确认具备本地目录的读写权限。
   - **行为**：自动维护 `03-掌握度账本.md`、知识图谱及日期记录。写入时严格遵守存储契约，仅在标记为 `AI-MANAGED` 的区域修改，不得随意污染用户手写内容。

2. **云笔记本/预览模式 (Cloud/Preview Mode)**：
   - **条件**：无直接本地读写权限，但用户表示使用云端工具或需要预览。
   - **行为**：不执行物理写入，也不报错中断。AI 应当将生成的 Ledger 更新、笔记卡片包装成结构化的 `[待写入]`、`[Notion Patch]` 或 `[飞书追加]` 的 Markdown 代码块输出，引导用户手动复制粘贴。

3. **纯会话模式 (Chat-only Mode)**：
   - **条件**：无任何持久化介质或工作区上下文。
   - **行为**：完全基于大模型会话上下文缓存。讲授与出题环路正常运转，但在每个会话结束时，AI 必须自动生成一个紧凑的 `[Session Handoff Block]`（会话状态移交块），要求用户自行保存，下次输入此块即可无缝恢复进度。

---

## 影响
- ** pedagogical engine 的轻量化**：教学评估和出题的逻辑不与文件读写强绑定。
- **用户体验**：不管学生在什么终端启动学习，都能即开即用，并在没有权限时提供极其顺畅的降级交互指引，不会因“无文件写入权限”而直接报错跳出。
