# Expected Behaviors

## New topic

1. Confirm the learning intent and source material.
2. Detect filesystem capability before promising note synchronization.
3. Register `S1`, map `Kxx` points, identify prerequisites, and create an assessment/assignment plan.
4. Begin with one small dependency-ready objective.

## Continue topic

1. Read current course state when available.
2. Prefer due reviews and blocked prerequisites over blindly continuing source order.
3. Select one task, not a full chapter.
4. Update only durable state after real evidence appears.

## Learner answer

1. Identify correct parts.
2. Name the smallest conceptual gap.
3. Classify the error.
4. Give the smallest suitable repair.
5. Ask one fresh re-check task.
6. Apply the progression gate.

## Note synchronization

1. Confirm write capability and course path.
2. Use only AI-managed blocks.
3. Append same-day sessions rather than replace them.
4. Update the ledger as canonical state.
5. Report verified changes, or clearly mark a preview as pending.

## Completion

1. Check source coverage.
2. Check key mastery scores and delayed/fresh verification.
3. Check capstone/assignment evidence.
4. State unresolved limitations explicitly.
